# Планувальник бюджету

Особистий планувальник місячного бюджету з ОВДП-портфелем, цілями накопичення
та опційною синхронізацією через Google Sheets (Apps Script).

Раніше застосунок жив одним файлом `index.html` (~4100 рядків, із вшитим
minified React і кодом у вигляді `React.createElement`). Тепер це звичайний
Vite-проєкт з модульною структурою.

## Запуск

```bash
npm install
npm run dev      # локальна розробка з hot-reload
npm run build    # збірка у dist/
npm run preview  # перегляд зібраної версії
```

## Структура

```
index.html              — HTML-оболонка (підключення шрифту Inter + точка входу)
src/
  main.jsx              — точка входу, монтує <App/>
  App.jsx              — головний компонент: бюджет, вкладки, синхронізація
  index.css            — усі стилі (включно з @media print для PDF)
  lib/
    constants.js       — місяці, ключі localStorage, шаблони, валюти, кольори,
                         константи ОВДП-домену
    budget.js          — правило 50/30/20: KIND_MAP, guessKind()
    investments.js     — логіка ОВДП: купони, групи виплат, міграції, критерії
    nbu.js             — отримання курсу валют з API НБУ
  components/
    Icon.jsx
    InvestmentsTab.jsx — вкладка ОВДП-портфеля
    BondForm.jsx       — форма додавання/редагування облігації
    InvSettingsModal.jsx
    TableMirror.jsx    — дзеркало стану таблиці Google Sheets
    SyncModal.jsx      — підключення URL синхронізації
```

## Розподіл бюджету: пріоритетні категорії

Замість статичного 50/30/20 від повного доходу застосунок використовує
динамічну модель «спочатку зафіксуй, потім розподіли залишок»:

1. Будь-яку категорію можна позначити **🔒 Пріоритет** і ввести фіксовану суму
   (наприклад, 50% зарплати на «Авто»).
2. Усі зафіксовані суми резервуються першими.
3. Решта доходу (`дохід − Σ зафіксованих`) ділиться між **незафіксованими**
   категоріями за пропорціями 50/30/20, перенормованими лише на ті типи
   (потреба/бажання/заощадження), де ще лишилися незафіксовані категорії.

Приклад: дохід 40 000, «Авто» зафіксовано на 20 000 → залишок 20 000
розподіляється: потреби 12 500, бажання 7 500 (заощадження вже покрите Авто).
Без жодної фіксації модель збігається з класичним 50/30/20 від повного доходу.

Логіка живе в `App.jsx` (`recommendedFor`, `availableAfterFixed`, `weightSum`),
поле `group.fixed` опційне і зберігається разом з місяцем.

## Живий залишок на кожну категорію

Під підсумком кожної категорії показується, **скільки доходу доступно саме
на неї** = `дохід − усе, що вже зайнято рештою` (інші категорії + заплановані
внески цілей). Цифра й тонкий прогрес-бар оновлюються живо: щойно вписуєш суму
деінде, доступне на цю категорію перераховується. Якщо введена сума перевищує
доступне, рядок підсвічується й показує, на скільки перебір.

Логіка: `availableForGroup` / `spentOutside` в `App.jsx`.

## Зберігання даних

- **localStorage** — основне сховище (ключі з префіксом `budget_v2`).
- **Google Sheets** (опційно) — снапшот усього стану через Apps Script
  (POST для запису, GET для читання). Таблиця вважається джерелом правди:
  при підключеному аркуші дані підтягуються з нього на старті.

## Примітки щодо міграції

Код компонентів поки що написаний у вигляді `React.createElement` (як було в
оригіналі) — це валідний JS, що працює без змін. JSX можна вводити поступово,
файл за файлом, без ризику зламати решту.
